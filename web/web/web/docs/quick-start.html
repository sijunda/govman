Quick Start Guide

Get up and running with GOVMAN in minutes. This guide will walk you through installation, configuration, and your first Go version management tasks.

Prerequisites

Before installing GOVMAN, ensure your system meets these requirements:

  • Operating System: Windows 10+, macOS 10.14+, or Linux (Ubuntu 18.04+, CentOS 7+)
  • Architecture: x86_64 (amd64) or ARM64
  • Shell: bash, zsh, fish, or PowerShell
  • Network: Internet connection for downloading Go versions
  • Disk Space: At least 500MB for GOVMAN and Go installations
💡 Note

GOVMAN does not require administrator privileges and installs entirely in your user directory for maximum security and convenience.

1

Installation

Choose the installation method that works best for your operating system:

macOS and Linux

bash 
# Using curl (recommended)
curl -sSL https://get.govman.dev/install.sh | bash

# Using wget (alternative)
wget -qO- https://get.govman.dev/install.sh | bash

Windows

powershell 
# PowerShell (Run as User)
iwr -useb https://get.govman.dev/install.ps1 | iex

# Command Prompt (alternative)
powershell -c "iwr -useb https://get.govman.dev/install.ps1 | iex"

Manual Installation

If you prefer manual installation or the automated scripts don't work for your system:

  1. Download the latest release from GitHub Releases
  2. Extract the archive to your preferred location
  3. Add the GOVMAN binary to your system PATH
  4. Run govman init to configure shell integration
⚠️ Shell Restart Required

After installation, restart your terminal or run source ~/.bashrc (or your shell's equivalent config file) to enable GOVMAN commands.

2

Verification

Verify that GOVMAN was installed correctly:

bash 
# Check GOVMAN version
govman --version

# View help information
govman --help

# Check installation status
govman status

You should see output similar to:

output 
GOVMAN v1.4.2
Platform: linux/amd64
Installation: /home/user/.govman
Shell Integration: ✓ Enabled
✅ Installation Successful

If you see version information, GOVMAN is ready to use! If you encounter any issues, check the troubleshooting guide.

3

Installing Your First Go Version

Now let's install and activate your first Go version with GOVMAN:

Install the Latest Go Version

bash 
# Install the latest stable Go version
govman install latest

# Alternative: install a specific version
govman install 1.21.5

Activate the Go Version

bash 
# Use the installed version globally
govman use 1.21.5 --global

# Alternative: use for current session only
govman use 1.21.5

Verify Go Installation

bash 
# Check Go version
go version

# Verify GOPATH and GOROOT
go env GOPATH GOROOT
💡 Global vs Session

Using --global makes the Go version persistent across terminal sessions. Without it, the version is only active for the current session.

4

Switching Between Versions

One of GOVMAN's most powerful features is the ability to quickly switch between different Go versions:

Install Multiple Versions

bash 
# Install multiple Go versions
govman install 1.21.5
govman install 1.20.12
govman install 1.19.13

# List all installed versions
govman list

Switch Between Versions

bash 
# Switch to different versions
govman use 1.21.5    # Session only
govman use 1.20.12   # Session only
govman use 1.19.13 --global  # Make it the default

# Check current active version
govman current
go version

List Available Versions

bash 
# List installed versions
govman list

# List all available versions (including not installed)
govman list --all

# List only latest versions
govman list --latest
⚡ Speed Comparison

Version switching with GOVMAN typically takes less than 1 second, compared to 10-30 seconds with traditional methods. This speed comes from smart caching and optimized PATH management.

5

Project-Specific Configuration

GOVMAN supports project-specific Go versions, allowing different projects to use different Go versions automatically:

Set Project Version

bash 
# Navigate to your project directory
cd /path/to/your/project

# Set Go version for this project
govman use 1.21.5 --local

# This creates a .govman-version file
cat .govman-version

Automatic Version Switching

With shell integration enabled, GOVMAN automatically switches to the project's Go version when you enter the directory:

bash 
# Enable shell integration (if not already done)
govman init

# Now when you enter a project directory:
cd /path/to/project-with-go-1.21
# GOVMAN automatically switches to Go 1.21.5

cd /path/to/project-with-go-1.20  
# GOVMAN automatically switches to Go 1.20.12

Manual Version File

You can also create the version file manually:

bash 
# Create version file manually
echo "1.21.5" > .govman-version

# Add to version control
git add .govman-version
git commit -m "Add Go version specification"
👥 Team Collaboration

Commit the .govman-version file to your repository so all team members use the same Go version automatically when they enter the project directory.
